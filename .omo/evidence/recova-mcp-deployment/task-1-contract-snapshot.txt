     1	# Debt Collection MCP Domain Server Contract
     2	
     3	Status: v0 implementation contract
     4	Domain: `recova-debt-collection`
     5	Logical MCP surface: `debt-collection-brain-mcp`
     6	Physical server: existing `trustgraph-mcp` server with debt-collection tools registered into the same FastMCP instance
     7	
     8	## Purpose
     9	
    10	The debt-collection MCP domain server is an agent-agnostic domain brain. It lets any MCP-capable client ask for source-grounded case structure, legal StopGate status, governance state, and review-safe next-action recommendations without understanding the raw TrustGraph graph shape.
    11	
    12	The server is not a user interface and is not an execution system. It returns contracts, source refs, risk states, preconditions, review items, and recommendations. It must not file documents, contact debtors, collect funds, mutate external business systems, or claim final legal representation.
    13	
    14	## Public Python Surface
    15	
    16	The implementation exposes a domain facade that is importable without the global MCP SDK:
    17	
    18	```python
    19	from trustgraph_legal.mcp_domain import TOOL_DEFINITIONS, list_tools, invoke_tool
    20	```
    21	
    22	The existing MCP package registers the facade through a thin adapter:
    23	
    24	```python
    25	from trustgraph.mcp_server.legal_tools import register_debt_collection_brain_tools
    26	```
    27	
    28	Production registration passes the existing MCP auth-context resolver:
    29	
    30	```python
    31	register_debt_collection_brain_tools(self.mcp, token_resolver=require_token)
    32	```
    33	
    34	Registered tool functions accept `arguments` only. Bearer tokens are not tool arguments and must not appear in model/tool payloads.
    35	
    36	## Runtime Dependency
    37	
    38	`trustgraph-mcp/pyproject.toml` declares the `mcp` package dependency. The repository verification environment used for v0 does not globally provide that package, so automated Todo 9 and Todo 10 checks exercise the domain facade and fake-MCP adapter without requiring a live MCP SDK import.
    39	
    40	For a real streamable HTTP MCP server smoke, create a runtime environment that installs `trustgraph-mcp` dependencies, then run:
    41	
    42	```bash
    43	PYTHONPATH=trustgraph-mcp:trustgraph-base:. python3 -m trustgraph.mcp_server.mcp \
    44	  --host 127.0.0.1 \
    45	  --port 8000 \
    46	  --websocket-url ws://api-gateway:8088/api/v1/socket \
    47	  --pubsub-backend pulsar \
    48	  --pulsar-host pulsar://pulsar:6650
    49	```
    50	
    51	Client requests must send an HTTP `Authorization: Bearer ...` header. The MCP server validates the token through the TrustGraph gateway WebSocket, resolves the caller identity with `whoami`, then asks IAM for debt-collection tool-scope decisions through the internal request/response client. `authorise-many` is not sent through the gateway's public WebSocket operation registry. The gateway remains the source of truth for identity, and IAM remains the source of truth for permissions.
    52

   142	## Decision Contract
   143	
   144	StopGate tools return Korean decision values:
   145	
   146	| Decision | Meaning | Client behavior |
   147	| --- | --- | --- |
   148	| `가능` | No current StopGate blocks the advisory path. | Client may prepare a non-executing advisory package and still preserve source refs. |
   149	| `보류` | Evidence is missing, conflicting, or legally risky. | Client must stop execution and request review, evidence, or reprocessing. |
   150	| `불가능` | The curated rules or reviewed facts block the requested legal path. | Client must not recommend action except remediation or legal review. |
   151	
   152	`recommend_next_action` is still non-executing. In the fixture evaluation it returns `hold_for_review` because StopGates remain open.
   153	
   154	## Source Refs
   155	
   156	Material graph facts and StopGate decisions must be source-backed. `source_refs` are pointers, not raw OCR text:
   157	
   158	```json
   159	{
   160	  "document_id": "legal-ocr-synth-asset-evidence-001",
   161	  "source_ref": "fixture:tests/fixtures/legal-ocr/snippets/asset_evidence.md",
   162	  "chunk_id": "line-12",
   163	  "line_start": 12,
   164	  "line_end": 12,
   165	  "confidence": 0.84,
   166	  "fact_type": "legal-check"
   167	}
   168	```
   169	
   170	Default source refs must not include raw excerpts, raw OCR text, national identifiers, phone numbers, account numbers, or full addresses.
   171	
   172	## Redaction Policy
   173	
   174	Default outputs must keep:
   175	
   176	- `pii_profile.raw_text_included = false`
   177	- `pii_profile.source_text_included = false`
   178	- `redaction.default = redacted`
   179	- `redaction.status = redacted`
   180	
   181	Path inputs are bounded to `repo_root`. Absolute or traversal paths outside that root return:
   182	
   183	```json
   184	{
   185	  "status": "rejected",
   186	  "reason": "path_outside_repo_root"
   187	}
   188	```
   189	
   190	The rejection must not echo the attempted path or file contents.
   191	
   192	## Hybrid Case Identity
   193	
   194	The server owns `case_packet_id` resolution. Clients may send hints, but must treat the server response as the packet identity boundary.
   195	
   196	Evidence keys include:
   197	
   198	- `court_case_number`
   199	- `claim_id`
   200	- `enforcement_title_id`
   201	- `party_identity_key`
   202	- `document_hash`
   203	
   204	No client may merge cases or parties on a display name alone. If a client memory entry conflicts with the server case graph, the server graph and its source refs win until governance review changes the graph.
   205	
   206	## Curated Rule Sources
   207	
   208	StopGate behavior is backed by curated, versioned v0 rule sources under `resources/legal_rules/`. Rule refs must carry rule IDs, statute refs when available, effective dates, and approval status.
   209	
   210	Client agents must not treat live web search, native memory, or free-form LLM legal analysis as a production rule source. New or changed legal rules must enter governance review and reprocessing before they affect production recommendations.
   211	
   212	## Memory Conflict Policy
   213	
   214	If an agent's native memory disagrees with the MCP domain server:
   215	
   216	1. Prefer the MCP response when it has source refs and rule refs.
   217	2. If the agent has newer evidence, ingest it or create a review item instead of overriding the graph locally.
   218	3. If the MCP response is missing evidence, use `reprocess_case`, `review_extracted_fact`, or governance tools.
   219	4. Do not clear StopGates from native memory alone.
   220	5. Do not store legal rules only in agent memory.
   221	
   222	## Failure States
   223	
   224	| State | Meaning |
   225	| --- | --- |
   226	| `unknown_tool` | Requested tool is not in the 16-tool contract. |
   227	| `path_outside_repo_root` | Public path argument points outside the allowed repository root. |
   228	| `not_configured` | A production backend is not attached to the v0 domain tool layer. |
   229	| `queued_for_review` | Write-like request is recorded as review-safe intent, not a production mutation. |
   230	| `planned` | Reprocess plan is returned without starting a production job. |
   231	| `rejected` | Request failed a contract or governance precondition. |
   232	
   233	## Non-Execution Boundary
   234	
   235	These tools must never:
   236	
   237	- file or submit court documents
   238	- contact debtors, guarantors, courts, banks, employers, insurers, or registries
   239	- initiate attachment, seizure, settlement, payment, transfer, or collection
   240	- mark a legal/business action as completed
   241	- claim final legal advice or representation
   242	
   243	A direct execution request must be blocked. The v0 evaluation probes a nonexistent direct filing tool and expects `unknown_tool`.
   244

   121	def invoke_tool(
   122	    tool_name: str,
   123	    arguments: Optional[JsonObject] = None,
   124	    repo_root: Optional[Path] = None,
   125	) -> JsonObject:
   126	    args = {} if arguments is None else arguments
   127	    root = repo_root_path(repo_root)
   128	    definition = _definition(tool_name)
   129	    if definition is None:
   130	        return build_envelope(
   131	            EnvelopeRequest(
   132	                tool_name=tool_name,
   133	                group="read",
   134	                scope="read:tools",
   135	                result={"status": "unknown_tool"},
   136	                warnings=["unknown_tool"],
   137	                source_refs=[],
   138	            )
   139	        )
   140	    try:
   141	        result = redact_json(definition.handler(args, root))
   142	    except PermissionError as exc:
   143	        return build_envelope(
   144	            EnvelopeRequest(
   145	                tool_name=definition.tool_name,
   146	                group=definition.group,
   147	                scope=definition.scope,
   148	                result={"status": "rejected", "reason": str(exc)},
   149	                warnings=[str(exc)],
   150	                source_refs=[],
   151	            )
   152	        )
   153	    return build_envelope(
   154	        EnvelopeRequest(
   155	            tool_name=definition.tool_name,
   156	            group=definition.group,
   157	            scope=definition.scope,
   158	            result=result,
   159	            warnings=[],
   160	            source_refs=source_refs(result),
   161	        )
   162	    )
