# Hermes 에이전트용 추심 두뇌 v0 부족한 점

현재 구조는 추심 업무용 온톨로지, 사건 그래프, StopGate, MCP 도구를 붙일 수 있는 기본 골격이다. 다만 아직 숙련 추심 담당자처럼 독립적으로 일하는 수준은 아니며, 아래 보강이 필요하다.

## 1. 법률 규칙 고도화

- 지급명령 확정 여부, 집행권원 효력, 승계집행문, 소멸시효 중단/완성 판단을 더 촘촘히 넣어야 한다.
- 회생, 파산, 면책, 압류금지채권, 급여압류, 채권압류, 부동산/동산 집행별 예외 규칙이 더 필요하다.
- 이자 제한, 지연손해금 계산, 추심금지 행위 같은 실무 위험 규칙도 추가해야 한다.

## 2. 실제 문서 대량 인제스트

- 지금은 계속 인제스트할 수 있는 입구가 열린 상태다.
- 실제 추심 문서들을 많이 넣어 문서 타입, 필드 추출, 사건 그래프 품질을 계속 높여야 한다.
- 모르는 문서나 애매한 문서는 `unknown`/review로 남기고, 검토 후 온톨로지 후보로 승격해야 한다.

## 3. 담당자 판단 피드백 루프

- 실제 담당자가 내린 판단과 에이전트 판단을 비교하는 루프가 필요하다.
- 틀린 이유를 규칙, 온톨로지, 추출기, StopGate 중 어디 문제인지 나눠 기록해야 한다.
- 반복적으로 수정해야 전문성이 올라간다.

## 4. 평가셋 구축

- 대표 케이스별 정답 세트를 만들어야 한다.
- 예: 진행 가능, 보류, 중단, 추가 문서 필요, 금액 불일치, 시효 위험, 면책 위험.
- 새 규칙이나 새 문서 타입을 넣을 때마다 이 평가셋으로 회귀 검증해야 한다.

## 5. 실행 도구는 아직 제한적

- 현재 MCP 도구는 판단, 조회, 추천 중심이다.
- 실제 신청서 작성, 제출, 외부 시스템 액션은 안전상 바로 열지 않는 것이 맞다.
- 실행형 도구를 열려면 권한, 감사 로그, 사람 승인, 실패 복구 절차가 먼저 필요하다.

## 결론

지금 만든 것은 Hermes 같은 에이전트에 붙일 수 있는 추심 전문 두뇌의 v0 골격이다. 보조자/검토자 수준으로는 의미 있게 작동할 수 있지만, 숙련 담당자 수준이 되려면 실제 문서, 법률 규칙, 담당자 피드백, 평가셋이 계속 축적되어야 한다.
